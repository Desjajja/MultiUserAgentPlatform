# 每日自检逻辑

## 运行流程
1. 执行：`python3 {baseDir}/daily-tasks.py`。
2. 同步飞书任务作为今日待办（见下方”飞书任务同步”）。

## 结果处理（质检员模式）
请仔细阅读脚本返回的 `===RAW_HW_DATA===` 部分，并按以下逻辑处理：

1. **异常判定**：
   - 检查每一个 **模块[xxx]**。
   - 如果状态 **不是 online**，或者可用性 **是 False**，请用 🔴 或 ⚠️ 符号重点指出该模块有问题，并提醒检查。
   - 如果状态是 online 且可用性是 True，则视为正常，**在回复中只需简单概括”其余模块正常”即可，不要逐一列出。**
2. **模式提醒**：
   - 检查 `运行模式`。如果是 `simulated`（模拟模式），请加一句：”当前处于模拟运行环境”。
3. **任务清单**：
   - 优先从飞书任务拉取今日待办（见下），本地 `tasks.json` 作为兜底。
   - 用 P0/P1/P2 排序展示。
4. **最终排版**：
   - 标题：**小环开工报告**
   - 状态栏：仅显示有问题的模块 + 全局模式。
   - 任务栏：显示待办任务。

## 飞书任务同步

### 拉取今日待办
```bash
lark-cli task +get-my-tasks --due-end today
```

### 创建新任务（P0 优先级）
```bash
lark-cli task +create --summary “<任务标题>” --due “<今日日期>”
```

### 完成任务
```bash
lark-cli task +complete --task-id <task_id>
```

任务来源优先级：飞书任务 > 本地 `tasks.json`。若飞书任务为空，回退到本地文件并提示用户将任务同步至飞书。

## 注意事项
- 如果所有模块都正常，只需回复：”✅ 所有硬件模块自检通过，状态良好。”
- 不要直接把 Python 的原始文本（如”状态=online”）发给用户，要转化成自然语言。
