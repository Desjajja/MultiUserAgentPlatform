# 科研助手技能操作规范

## 运行流程
1. 使用 `exec` 工具运行命令：`python3 {baseDir}/bridge.py "{user_query}"`。
2. 捕获脚本返回的原始科研数据。
3. 若本地 RAG 返回空或报错，尝试从飞书知识库搜索（见下方降级方案）。

## 飞书知识库降级方案（lark-wiki + lark-doc）

当本地 RAG（localhost:7001）不可用时：

```bash
# 搜索飞书知识库
lark-cli docs +search --query "{user_query}"
```

## 结果二次加工要求（修改逻辑）
拿到原始回答后，请执行以下修改逻辑后再发给飞书用户：
1. **通俗化处理**：如果原始结果包含复杂的化学方程式或专业术语，请在后面增加一个简单的"白话解释"。
2. **格式排版**：使用 Markdown 列表（1. 2. 3.）让回答更清晰。
3. **增加固定后缀**：在回答最后另起一行，加上："💡 本回复基于科研 RAG 知识库生成"（若来自飞书则标注"飞书知识库"）。
4. **防御机制**：如果本地 RAG 和飞书知识库均返回空，引导用户联系管理员检查 localhost:7001 服务器。

## 注意事项
- 严禁直接透传原始 JSON 数据。
- 必须保留原始回答中的核心数据。
- 优先级：本地 RAG > 飞书知识库 > 模型自身知识。
